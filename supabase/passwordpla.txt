Include Password in Team Invite Email
Problem
When a team member is invited, Supabase's inviteUserByEmail() sends a default email with only a magic link. The temporary password generated by the edge function is never emailed to the new user.

Proposed Changes
[MODIFY] 
index.ts
After creating the user and setting the password, send a custom email via the Resend API (free tier: 100 emails/day) that includes:

Welcome message with company name
Login email + temporary password
Login URL
Prompt to change password on first login
Key changes:

Look up the company name from the companies table for branding
Call Resend API to send a formatted HTML email to the new user
Only send for new users (existing users keep their current password)
IMPORTANT

You will need to:

Sign up at resend.com and get a free API key
Add the secret to Supabase: Dashboard > Edge Functions > Secrets > Add RESEND_API_KEY
Verify a sender domain or use the default onboarding@resend.dev for testing
Verification Plan
Manual Verification
Add the RESEND_API_KEY secret in Supabase Dashboard
Deploy the updated function via Dashboard
Add a new team member from the app
Check the new member's email for the welcome message with credentials
